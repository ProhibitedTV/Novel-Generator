# Releasing

## Versioning

This project uses semantic versioning once public releases begin. Until `1.0.0`, breaking changes may happen between minor releases.

## Release Checklist

1. Update the version in `pyproject.toml` and `src/novel_generator/__init__.py`.
2. Run the test suite.
3. Review docs for any setup or API changes.
4. Tag the release in git.
5. Publish release notes describing new features, fixes, and any migration notes.

## Suggested Release Notes Sections

- Highlights
- Fixes
- Deployment notes
- Breaking changes
