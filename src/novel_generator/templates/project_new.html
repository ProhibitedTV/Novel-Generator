What happens after this

Creating a project is the handoff from setup into actual work.

  1. 1
    Save the project defaults

    This keeps your premise, chapter targets, and preferred model together for later runs.

  2. 2
    Queue a full draft run

    The worker should outline the book, draft every requested chapter, save summaries, then export artifacts.

  3. 3
    Manage finished runs later

    Project pages keep rerun, cleanup, and deletion controls nearby once you start accumulating drafts.
