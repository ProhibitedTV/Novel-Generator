What a v2 run produces

A healthy run should finish every requested chapter, run editorial QA, and export a manuscript plus QA report.

  1. 1
    Build a story bible

    The model turns your premise and story brief into theme, cast, world rules, and an ending promise.

  2. 2
    Generate a structured outline

    Each chapter gets an objective, conflict turn, reveal, and ending state.

  3. 3
    Draft and revise every chapter

    Each chapter gets a plan, prose, one critique pass, and a continuity update checkpoint.

  4. 4
    Run editorial QA before export

    The QA report calls out repetition, continuity drift, and ending issues instead of silently shipping them.
