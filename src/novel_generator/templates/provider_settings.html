    How routing works

    Keep the default simple, then route only the expensive or editorial stages elsewhere when it actually helps.

    Recommended setup flow

    This keeps the first-run experience predictable.

    1. 1
      Keep Ollama as the default

      Use a verified Ollama model for the whole run unless you have a clear reason to route a specific stage elsewhere.

    2. 2
      Enable extra providers only if needed

      Optional providers are best used for story bible, outline, critique, revision, or QA when you want a different tradeoff.

    3. 3
      Set per-task routing at the project level

      Projects can keep one provider/model for everything or override only selected stages.

    4. 4
      Review stage usage in the run monitor

      Run events and run details show which provider and model were used at each stage.
